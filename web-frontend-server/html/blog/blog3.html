Bài 3: Tìm hiểu về Nginx Reverse Proxy

Reverse Proxy là gì?

Reverse Proxy là một máy chủ đứng trước các web server, chuyển tiếp các yêu cầu của máy khách (client) đến các web server đó. Trong bài này, Nginx đóng vai trò là cổng vào duy nhất (Gateway).

Cân bằng tải (Load Balancing)

Chúng ta đã cấu hình Nginx để phân phối lưu lượng truy cập giữa 2 Web Server:

upstream web_cluster {
    server web-frontend-server:80;
    server web-frontend-server-2:80;
}

Thuật toán sử dụng là Round Robin, giúp hệ thống không bị quá tải cục bộ tại một server duy nhất, tăng tính sẵn sàng cho ứng dụng.
